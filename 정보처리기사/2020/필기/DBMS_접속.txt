<DBMS 접속>

1. DBMS 접속 기술의 개요
	- 응용 시스템은 사용자로부터 매개 변수를 전달받아 SQL을 실행하고 DBMS로부터 전달받은 결과를 사용자에게 전달.
	- 웹 응용프로그램은 웹 응용 시스템을 통해 DBMS 접근
	- 웹 응용 시스템은 웹 서버와 웹 애플리케이션 서버(WAS)로 구성
		사용자 <-> 웹 서버 <-> WAS <-> DBMS
		

2. DBMS 접속 기술
	- DBMS에 접속하기 위해 사용하는 API 또는 프레임워크를 의미
		API : 응용 프로그램 개발 시 운영 체제나 DBMS 등을 이용할 수 있도록 규칙 등에 대해 정의해 놓은 인터페이스
		프레임워크 : 소프트웨어에서는 특정 기능을 수행하기 위해 필요한 클래스나 인터페이스 등을 모아둔 집합체
	
	- JDBC
		> Java 언어
		> 접속하려는 DBMS에 대한 드라이버 필요
		> Java SE에 포함
		
	- ODBC
		> MS에서 출시
		> 개발 언어와 상관 없음
		> 다양한 데이터 베이스에 접근 가능
		
	MyBatis
		> JDBC 코드를 단순화하여 사용할 수 있는 SQL Mapping 기반 오픈소스 접속 프레임워크
		> SQL 문장을 분리하여 XML을 파일을 만들고 Mapping을 통해 SQL을 실행
		> SQL을 거의 그대로 사용할 수 있어 국내 환경에 적합
		
3. 동적 SQL
	- 개발 언어에 삽입되는 SQL코드를 문자열 변수에 넣어 처리하는 것.
